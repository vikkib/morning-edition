VentureBeat, Wednesday, May 27, 2026

On May 19, attackers passed 633 malicious npm package versions through Sigstore provenance verification by using a stolen maintainer account. The security system worked perfectly, validating the signatures and certificates. The attack succeeded precisely because the attacker had legitimate credentials.

This is a crucial lesson for any development work or client integrations you touch. No security system is stronger than credential management. It reinforces why access control and account hygiene matter more than the fanciest verification tools.
